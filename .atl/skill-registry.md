## How to use

When launching a sub-agent that needs a specific skill, the orchestrator MUST pass the
absolute file path to the skill's `SKILL.md` in the prompt, e.g.:

```
SKILL: Load `C:\Users\camil\.config\opencode\skills\sdd-spec\SKILL.md` before starting.
```

Sub-agents do NOT search for the skill registry themselves. They receive the path inline.

## Resolution rules

1. Project-level skills (`.claude/skills/`, `.gemini/skills/`, `.agent/skills/`, `skills/`)
   override user-level skills with the same name.
2. The first directory that contains a `<name>/SKILL.md` wins.
3. `sdd-*` and `skill-registry` are SDD orchestration skills and are NOT listed here —
   they're resolved implicitly by the orchestrator's SDD workflow.
